Getting Started

Welcome to the Cartify API documentation. Our API follows RESTful principles and returns JSON responses. To get started, you'll need an API key which you can obtain from your developer dashboard.

GET /status

Check API status and version information.

Request
Response
curl -X GET https://api.Cartify.com/v1/status \ -H "Authorization: Bearer YOUR_API_KEY"
{ "status": "ok", "version": "1.2.3", "timestamp": "2023-06-15T10:30:00Z" }

Authentication

All API requests require authentication using your API key. Include the key in the Authorization header of your requests.

POST /auth/login

Authenticate a user and receive an access token.

Parameter Type Required Description
email string Yes User's email address
password string Yes User's password
Request
Response
curl -X POST https://api.Cartify.com/v1/auth/login \ -H "Content-Type: application/json" \ -d '{ "email": "user@example.com", "password": "password123" }'
{ "success": true, "token": "eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9...", "user": { "id": 12345, "email": "user@example.com", "name": "John Doe" } }

Products

Retrieve and manage product information in the Cartify catalog.

GET /products

Retrieve a list of products with optional filtering and pagination.

Parameter Type Required Description
page integer No Page number for pagination (default: 1)
limit integer No Number of products per page (default: 20, max: 100)
category string No Filter by category ID
search string No Search term to filter products
Request
Response
curl -X GET "https://api.Cartify.com/v1/products?page=1&limit=10&category=electronics" \ -H "Authorization: Bearer YOUR_API_KEY"
{ "products": [ { "id": 101, "name": "Smartphone X", "description": "Latest smartphone with advanced features", "price": 699.99, "category": "electronics", "stock": 50, "images": ["https://..."] } ], "pagination": { "page": 1, "limit": 10, "totalPages": 5, "totalProducts": 48 } }

Categories

Manage product categories for organizing your catalog.

GET /categories

Retrieve all product categories.

Response: Returns an array of category objects with id, name, and description.

Orders

Create and manage customer orders.

POST /orders

Create a new order.

Parameter Type Required Description
items array Yes Array of product items with productId and quantity
shippingAddress object Yes Shipping address details
paymentMethod string Yes Payment method (credit_card, paypal, etc.)
Response: Returns the created order object with order ID and status.

Users

Manage user accounts and profiles.

GET /users/{id}

Retrieve a specific user's information by ID.

Parameter Type Required Description
id integer Yes User ID in the URL path
Response: Returns user object with id, name, email, and profile information.

Shopping Carts

Manage user shopping carts and cart items.

GET /carts/{userId}

Retrieve a user's shopping cart.

Response: Returns cart object with items, quantities, and total price.

Payments

Process payments and manage payment information.

POST /payments

Process a payment for an order.

Parameter Type Required Description
orderId integer Yes ID of the order to be paid
paymentMethod string Yes Payment method (credit_card, paypal, etc.)
paymentDetails object Yes Payment method-specific details
Response: Returns payment confirmation with transaction ID and status.

Webhooks

Set up and manage webhooks for real-time notifications.

POST /webhooks

Register a new webhook endpoint.

Response: Returns webhook configuration with ID and status.

Error Handling

Understand API error responses and status codes.

The API uses conventional HTTP response codes to indicate success or failure.

Code Description
200 Success
400 Bad Request - Invalid parameters
401 Unauthorized - Invalid API key
404 Not Found - Resource doesn't exist
429 Too Many Requests - Rate limit exceeded
500 Internal Server Error

Rate Limiting

Understand API usage limits and quotas.

The API is rate limited to 1000 requests per hour per API key. Additional limits may apply to specific endpoints.

Headers: Rate limit information is included in response headers (X-RateLimit-Limit, X-RateLimit-Remaining, X-RateLimit-Reset).
